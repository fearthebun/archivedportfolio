Kynab is an author that writes fantasy fiction. She writes in a very strange manner: she puts her plot on one page. Setting, character development, and interactions happen somewhere else. Let’s watch her write “The Magical Adventure,” her four hundred and twentieth book. First, Kynab needs a setting. She grabs a binder she’s titled “Magical Kingdoms” and reads the first bit:

Every Magical Kingdom has a few key ingredients: forests, castles, dragons, unicorns, princesses, and princes. A standard Magical Kingdom would have one prince and one princess, one dragon, one castle located somewhere in Fantasyland and take place in 1200.

This description of a Magical Kingdom established the key ingredients for a Magical Kingdom and gave several values that might occur in a default magical kingdom.

MagicalKingdom

has a certain number of forests

has a certain number of castles

has a certain number of dragons

has a certain number of unicorns

has a certain number of princes

has a certain number of princesses

is located somewhere

exists at a certain time

a typical MagicalKingdom would have

number of castles = 1

location = Neverland

time period = 1200

The first line on the page states that Kynab is defining a Magical Kingdom. Kynab doesn’t use spaces between two words so that her editor is not confused. Next she declares all the variables (something that defines what the kingdom looks like) that a class of magical kingdoms has. Lastly, she needs to construct a sample kingdom with default values on all her variables.

public class MagicalKingdom

{

int forests;

int castles;

int dragons;

int unicorns;

int princes;

Int princesses;

String location;

int time;

public MagicalKingdom()

{

castles = 1;

location = "Fantasyland";

time = 1200;

}

}

To make her kingdom, Kynab defines a class, not the actual kingdom (which would be an object). The class says what elements are in a Magical Kingdom, but not their amounts. She is declaring the variables - what they will be called, and what they will store. She uses a semicolon to let her editor know she’s done with a statement and curly braces to group sets of statements. When Kynab instantiates a kingdom object in her book, her editor will check the Magical Kingdoms class to see what the kingdom object will look like - but to discover the amounts and specifics, they will need the constructor. After all the variables have been declared, Kynab writes the constructor. It is called MagicalKingdom(), and contains the default values for her variables, or how Kynab’s kingdom object would look if she instantiated it without specifying anything.

Now, Kynab begins to write the first part of the plot.

The Magical Adventure

This story takes place in a Magical Kingdom called Fantasyland and has three forests, one dragon, one unicorn, one prince, and one princess.

When Kynab sends the story to her editor, he will look up “Magical Kingdom” and find out several pieces of information such as the number of castles. Kynab has also added new information about the dragons, princes, and unicorns, which her editor knew the magical kingdom would contain, but not the number of them.

Main Magical Adventure

Magical Kingdom fantasyLand is a new Magical Kingdom.

the number of forests in fantasyLand is three.

the number of dragons is one.

the number of unicorns is one.

the number of princes is one.

the number of princesses is one.

First, Kynab instantiates an object of type MagicalKingdom. Having created her first object, Kynab then goes on to fill out the information that was initially lacking in the MagicalKingdom class.

public class MagicalAdventure {

public static void main (String arguments[]) {

MagicalKingdom fantasyLand = new MagicalKingdom;

fantasyLand.forests = 3;

fantasyLand.dragons = 1;

fantasyLand.unicorns = 1;

fantasyLand.princes = 1;

fantasyLand.princesses = 1;

}

}

Kynab lets her editor know that this piece of paper is her main routine, or plot. When she decides to specify those variables (forests, etc) that were not specified in the MagicalKingdom class, Kynab first states the name of the object (in this case, fantasyLand, because it is an object of MagicalKingdom) followed by a period and then by the name of the variable. Having done this, Kynab can then provide a value for the variable, such as two.

Kynab first created a class (or the binder) that described a Magical Kingdom, then on her main plot instantiated a Magical Kingdom object called fantasyLand. It’s time to move on to her villain, so she looks for her binder labelled “Creatures.” It says:

"All creatures start out with 4 limbs, 2 eyes, a nose, and a mouth. They are of a certain species, have a name, are a certain colour, and have a magic power. If someone asks, creatures can respond with their name."

Kynab likes to make all body parts variables. This gives her the freedom to change their values later in the story, in case any of her characters are shapeshifters. She left the species, name, colour, and magic power unspecified, so that she can specify those variables in her main plot.

Creatures

have a magic power

Notice that Kynab's creature has some interactivity. She can ask its name, and it will respond. This is called a method.

public class Creatures {

int limbs;

int eyes;

int nose;

int mouth;

String name;

String species;

String colour;

String power;

public Creatures() {

limbs = 4;

eyes = 2;

nose = 1;

mouth = 1;

}

public String askName() {

return name;

}

}

Flipping through the "Creatures" binder, Kynab comes to the first chapter, entitled "Dragons."

Dragons are based on the idea of creatures. They are identical, except that they have some additional qualities, namely a wings, horns, a tail, a certain "look", some level of rage, and a certain quantity of coins in their possession. Your standard dragon will look mean, start the day out not angry, and not yet have stolen any coins from unicorns.

Kynab's binders (classes) are divided into subsections (subclasses). This particular subsection, dragons, extended the idea of creatures.

Dragons extends the idea of Creatures.

A Dragon

has wings.

has horns.

has a "look".

has some level of rage.

has a certain number of coins stolen.

For a given Dragon,

He will look mean.

He will start out not angry.

He will start the day without having stolen any coins.

public class Dragon extends Creatures {

String wingColour;

int numberOfCoins;

Creatures unicorns;

public Dragon() {

look = "Mean";

anger = 0;

numberOfCoins = 0;

}

}

We have introduced here the idea of subclasses. "Creatures"; was a class, and "Dragons" a subclass of it. Kynab also declared a “Creatures unicorns” in this class - if the dragon is going to steal coins from a unicorn, Kynab's editor has to know what a unicorn is. By declaring "Creatures unicorns", her editor will know that a unicorn is a type of creature.

After Kynab had described what her dragon looked like, she decides to move on to some of the dragon’s methods. Kynab wants to focus on how dragons become angry. She writes:

Whenever the main plot says that a dragon gets taunted, his level of anger will go up by one.

Kynab is specifying how one of her characters acts, as opposed to how they look. She is also altering one of her variables (the variable “anger”).

getTaunted

anger increases by one

In this method, Kynab treats variables very similarly to the past. She writes what she’s describing (a method called getTaunted), then what the method will do when she calls it in her main. 

public void getTaunted() {

anger++;

}

The first thing Kynab did was name the method "getTaunted". Kynab modifies the dragon’s level of anger, in this case by adding one to it. That is what the "++" after "anger" does.

Kynab decides that it's a good idea to allow her dragon to state how angry he is, so, she writes a new method:

If someone asks a dragon how angry he is, the dragon will always respond with his level of anger.

howAngryAmI

tell them how angry I am

public int howAngryAmI() {

return anger;

}

Kynab put “int” instead of “void” because this method has a return value. She said “int” for integer so her editor knows what type of value is being returned.  

Kynab decides to have the villain have the ability to steal coins. She sets out to write one more method. What makes this different than her other methods is that it needs information about someone other that the dragon: the unicorn he steals from. To allow for flexibility (and for a variety of unicorns), Kynab decides to leave the identity of the unicorn blank for now.

If the dragon is supposed to steal coins from a unicorn, steal the coins from the specified unicorn, then add one to the number of coins he has stolen. Then print out "The dragon has stolen from (the specified unicorn)!"

Kynab can leave certain things to be specified only when the plot is written (by writing that the unicorn will have a name, but not stating what it will be). She has also used a print statement that will print out the information gathered by the method as an event in her book.

stealCoin (name)

add one to the number of coins this dragon has stolen.

print "The Dragon has stolen from (the specified unicorn)!”

public void stealCoin (Creatures unicorns) {

this.unicorns = unicorns;

numberOfCoins++;

System.out.println("The Dragon has stolen from " + unicorns.askName() + “!”);

}

First, there is the name of the method: "stealCoin". Then, between the parentheses, Kynab writes "Creatures unicorns". This is called an argument, and it helps make the method more specific. When Kynab starts writing her plot, she will at some point want her dragon to steal from a unicorn. Kynab wants to add the name of the unicorn who was robbed. Hence, when her plot says "stealCoin", it will also say the name of the unicorn being stolen from as an argument. "Creatures", which comes before "unicorns", tells her editor that a creature is being stolen from.

Even though she has supplied the name of the unicorn as an argument, her editor won't let her use it in the method until she has stated that the "unicorn" that she declared as a "Dragons" variable is equal to the value given in the method's argument.

Once the dragon has stolen coins, the variable "coinsStolen" will surely go up by one, so Kynab puts the same "++" after coinsStolen that she had put after "anger" in the "getTaunted" example.

The "System.out.println" statement tells her editor know that what comes between those parentheses goes into the final text of the book: all the text she puts in between quotation marks. The variable unicorn.askName is not in quotation marks because it is an object's method. By keeping it out of the quotation marks, Kynab lets her editor know that he has to go look for the unicorns object (he would find out that it was of the "Creatures" class) then for the method "whatIsYourName" (he would see that it returned the unicorn’s name).

Kynab decides to work on her plot, or her main routine.

When Kynab created that town called Fantasyland, she created (instantiated) an object of type Magical Kingdom. This object had all the characteristics of a standard Magical Kingdom. The same thing goes for the Bartholomew The Second Fluffy Destroyer of Worlds object and the Princess Magic Marigold Pony object. While some of the traits of these objects were specified at the time of creation, most of their traits were specified back in the binders, or classes. This allowed Kynab to say a good deal while only saying a little bit in her main routine. Kynab's main plot (routine) is a collection of references to objects, which in turn are references to classes. Everything is dealt with at the lowest level possible.

The benefits of this are only needing to make changes in one place, instead of in many. If Kynab’s editor wanted a book of five hundred pages, and Kynab decided Bartholomew The Second Fluffy Destroyer of Worlds should have black horns, it isn’t hard to change it. Instead of changing every instance where they mentioned Bartholomew the Second Fluffy Destroyer of Worlds having gold horns, she could just change the Bartholomew the Second Fluffy Destroyer of Worlds object at the time of instantiation. Or she could change the Dragon class itself to only have black horns, or create a new class of dragons with black horns.  As well, there is the potential for interactivity like this.

Kynab’s editor reads Kynab’s manuscripts and then returns them with all the errors she made.

On the computer, this is called a compiler. All Java programs must be compiled before they can be run. The compiler will tell you everything that you did wrong, so you can go back and try to fix it.
